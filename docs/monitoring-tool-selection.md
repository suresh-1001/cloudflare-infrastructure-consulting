# Monitoring Tool Selection Framework

## Core Categories
- Uptime monitoring
- Synthetic transaction monitoring
- Infrastructure metrics
- Log aggregation
- Error tracking
- Alert routing and escalation

## Evaluation Criteria
- Ease of deployment
- Alert quality
- Noise reduction
- Dashboard clarity
- Cost
- Integration support
- Fit for current team maturity

## Typical Stack Model
- Uptime tool for external availability
- Performance/synthetic checks for user experience
- Metrics platform for infrastructure visibility
- Logging/error visibility for root cause analysis
- Alerting workflow tied to ownership and severity
